# Getting Started

Goal: go from zero to a first working chat with minimal setup.

:::note
Fastest chat: open the Control UI (no channel setup needed). Run `remoteclaw dashboard`
and chat in the browser, or open `http://127.0.0.1:18789/` on the

<abbr title="The machine running the RemoteClaw gateway service.">gateway host</abbr>. Docs:
[Dashboard](/web/dashboard) and [Control UI](/web/control-ui). :::

## Prereqs

- Node 22 or newer

:::tip
Check your Node version with `node --version` if you are unsure.
:::

## Quick setup (CLI)

<Steps>

1.  **Install RemoteClaw (recommended)**

                    <Tabs>
                      <TabItem label="macOS/Linux">
                        ```bash
                        curl -fsSL https://remoteclaw.org/install.sh | bash
                        ```
                        <img

                src="/assets/install-script.svg"
                alt="Install Script Process"
                className="rounded-lg"
                />

                </TabItem>

            <TabItem label="Windows (PowerShell)">

        `powershell
        iwr -useb https://remoteclaw.org/install.ps1 | iex
        `

        </TabItem>

    </Tabs>

                    :::note
                    Other install methods and requirements: [Install](/install).

                :::

2.  **Run the onboarding wizard**

    ```bash
    remoteclaw onboard --install-daemon
    ```

    The wizard configures auth, gateway settings, and optional channels.
    See [Setup](/start/setup) for details.

3.  **Check the Gateway**

    If you installed the service, it should already be running:

    ```bash
    remoteclaw gateway status
    ```

4.  **Open the Control UI**

    ```bash
    remoteclaw dashboard
    ```

</Steps>

:::tip[Success]
If the Control UI loads, your Gateway is ready for use.
:::

## Optional checks and extras

  <details>
<summary>Run the Gateway in the foreground</summary>

    Useful for quick tests or troubleshooting.

    ```bash
    remoteclaw gateway --port 18789
    ```

</details>
  <details>
<summary>Send a test message</summary>

    Requires a configured channel.

    ```bash
    remoteclaw message send --target +15555550123 --message "Hello from RemoteClaw"
    ```

</details>

## Useful environment variables

If you run RemoteClaw as a service account or want custom config/state locations:

- `REMOTECLAW_HOME` sets the home directory used for internal path resolution.
- `REMOTECLAW_STATE_DIR` overrides the state directory.
- `REMOTECLAW_CONFIG_PATH` overrides the config file path.

Full environment variable reference: [Environment vars](/help/environment).
